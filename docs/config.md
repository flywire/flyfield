# Configuration and Constants for flyfield

This document describes configuration settings, constants, and customizable parameters available in **flyfield** to tailor processing and output.

---

## Configuration Overview

flyfield uses a combination of CLI flags, environment variables, and internal constants to control behavior during extraction, field generation, filling, and capturing.

---

## CLI Configuration Options (Brief)

Common CLI options for controlling behavior include:

| Option               | Description                                       |
|----------------------|---------------------------------------------------|
| `--input-pdf FILE`   | Specifies the input PDF file.                     |
| `--pdf-pages PAGES`  | Specifies page numbers or ranges to process.      |
| `--input-csv FILE`   | Specifies CSV file for input or captured data.    |
| `--markup`           | Generate a visual markup PDF for detected fields. |
| `--fields`           | Generate interactive form fields in PDF.          |
| `--fill`             | Fill interactive fields with data from CSV.       |
| `--capture`          | Capture data from filled PDF back to CSV.         |
| `--debug`            | Enable detailed debug information and output.     |

---

## Internal Configuration Constants

Note: Detection only applies to **pure white vector boxes** matching size thresholds.

| Parameter                | Description                                   | Default Value          |
|--------------------------|-----------------------------------------------|------------------------|
| `MIN_BOX_HEIGHT`         | Minimum height (in points) for detected boxes | 10                     |
| `MIN_BOX_WIDTH`          | Minimum width (in points) for detected boxes  | 20                     |
| `MAX_BOX_HEIGHT`         | Maximum height for detected boxes              | 80                     |
| `MAX_BOX_WIDTH`          | Maximum width for detected boxes               | 300                    |
| `BOX_FILL_COLOR`         | Required fill color (RGB) for detection        | Pure white (255,255,255)|
| `FIELD_CODE_PREFIX`      | Prefix for autogenerated field codes           | Document-page-line-block naming scheme |
| `CSV_ENCODING`           | Encoding expected for CSV input/output          | UTF-8                  |

Extraction runs automatically unless an input CSV overrides the detection step.

## Customizing Field Detection

Customization can be done at the Python API level or by modifying the generated scripts:

- Adjust filtering criteria such as box sizes or colors to tune detection sensitivity.  
- Override default field naming convention for unique or legacy integration requirements.  
- Modify CSV field formats or add custom metadata columns as needed.

---

## Environment Variables

flyfield respects the following environment variables to influence runtime:

| Variable                         | Description                                |
|---------------------------------|--------------------------------------------|
| `flyfield_LOG_LEVEL`       | Sets logging verbosity (`DEBUG`, `INFO`, `WARNING`, `ERROR`) |
| `flyfield_CONFIG_FILE`     | Path to an explicit YAML or JSON config file (future feature) |

---

## Example: Adjust Minimum Box Size in Python Script

```

from flyfield import filter_fields

# Load detected fields list `fields`

# Filter out boxes smaller than 15x10 points

fields = filter_fields(fields, min_size=(15, 10))

```

---

## Notes

- Most users can operate flyfield effectively using CLI flags without modifying constants.  
- Advanced users and developers can customize processing by editing Python scripts generated during CLI operations.  
- Future versions may add support for config files for easier persistent configuration.

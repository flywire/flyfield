Glossary of Terms for flyfield

This glossary defines key terms and concepts used in the flyfield documentation and tooling.

Key Terms

AcroForm
A PDF form specification introduced in PDF 1.2 that enables interactive forms within PDF documents. Forms created with AcroForm contain elements like text fields, checkboxes, and buttons.

Field (Form Field)
An interactive area in a PDF where users can enter or select data, such as text boxes, checkboxes, radio buttons, and drop-down menus.

Field Code
A unique identifier automatically generated by flyfield for each detected white box in a PDF. Used to map form fields to CSV data.

Filled PDF
A PDF form that has been completed with data, either manually by a user or programmatically by flyfield filling scripts.

Interactive Field
A form field in a PDF that can be edited or filled using a PDF viewer or programmatically.

Markup PDF
A generated PDF that visually annotates detected white box placeholders and field codes to help verify detection and placement accuracy.

Placeholder Box (White Box)
A rectangular area in a vector PDF identified as a space where data should be entered. flyfield detects these based on color and size thresholds.

Working Copy PDF
A duplicate of the original source PDF used as a working file for detection, field generation, filling, and capturing steps. Working copies prevent modification of the original and help organize outputs.

PyMuPDF
A Python library used by flyfield for reading and manipulating PDF documents and their graphical elements.

PyPDFForm
A Python library utilized to create interactive PDF form fields programmatically.

Raster PDF
A PDF consisting primarily of scanned images or bitmap graphics rather than vector graphics, which flyfield cannot process effectively.

Vector PDF
A PDF that uses vector graphics (lines, shapes) for rendering, required by flyfield for detecting white boxes accurately.

Common Terms

BBox (Bounding Box)
The rectangle defined by coordinates that enclose an element such as a form field or graphical object in a PDF.

CSV (Comma-Separated Values)
A simple text format used to store tabular data. flyfield uses CSV files to import/export data for filling and capturing form fields.

Field Generator Script
A Python script generated by flyfield that creates PDF form fields based on detected white boxes.

Field Filler Script
A Python script generated to fill interactive PDF fields using data from CSV files.

Accessibility Terms

Tab Order
The sequence in which interactive fields receive focus when navigating a PDF form using keyboard input, important for usability.

Screen Reader
Assistive technology that reads the content of a PDF aloud to users with visual impairments.

Related Concepts

PDFBox
An open-source Java library for working with PDF documents, forming much of the foundation underneath flyfield.

AcroForm vs. XFA
Two types of PDF form technology. flyfield primarily works with AcroForm-based PDFs.

For further explanations and usage, see the relevant sections in the User Guide, API Documentation, and Quick Start Guide.
